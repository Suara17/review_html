## 每组整理原则
### 1. Python
- 保留：基础语义、内存模型、拷贝、装饰器、迭代器/生成器、GIL、进程线程协程、垃圾回收、常见容器对比。
- 精简：冷门库罗列、过细实现细枝末节。

### 2. MySQL
- 保留：索引、B+树、聚簇/非聚簇、回表/覆盖索引、事务、隔离级别、MVCC、锁、日志、慢查询优化。
- 精简：过细参数、版本差异中的非高频边角。

### 3. Redis
- 保留：常用数据结构、持久化、缓存击穿/穿透/雪崩、主从复制、哨兵、分片、分布式锁。
- 精简：较少问到的命令细节。

### 4. 计算机网络
- 保留：TCP/UDP、三次握手四次挥手、拥塞控制、HTTP/HTTPS、输入URL到页面展示、Cookie/Session。
- 精简：协议字段级冷门细节。

### 5. 操作系统
- 保留：进程线程协程、上下文切换、死锁、分页分段、虚拟内存、调度算法、用户态内核态。
- 精简：偏冷门历史知识。

### 6. 设计模式
- 保留：单例、工厂、代理、装饰器、观察者、策略、责任链。
- 精简：很少被直接考的模式长篇解释。

### 7. Git
- 保留：工作区/暂存区/仓库、merge/rebase、reset/revert、pull/fetch、冲突处理。
- 精简：命令选项大全。

### 8. 系统设计
- 保留：通用答题框架、高可用、高并发、缓存、消息队列、分库分表、限流熔断。
- 精简：过度抽象表述。

### 9-10. 项目面试类
- 保留：项目背景、目标、架构、关键难点、个人贡献、优化结果、追问口径。
- 精简：零散过程记录。

## 交付方式
- 采用“一个组一个组整理”的方式推进。
- 每整理完一组：
  1. 先生成精简版文稿
  2. 再回写对应 `topics-data/*.json`
  3. 如有需要，再重新生成对应 HTML 页面
